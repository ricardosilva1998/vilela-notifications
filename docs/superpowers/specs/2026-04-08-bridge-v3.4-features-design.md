# Bridge v3.4+ Feature Design — New Overlays + Improvements

## Context
Users want more data in existing overlays (tire compound, pit count, gap to leader) and two new overlays (pit strategy, lap comparison). Wind arrow has stutter issues. Track map needs size configurability. Session laps need tabs for P/Q/R separation.

## Phased Implementation

### Phase 1 — Quick Fixes + Data Foundation

#### 1a. Wind Arrow Smoothing
**File:** `bridge/overlays/wind.html`

The lerp factor (0.12) can cause visible steps when updates arrive at irregular intervals. Fix: use delta-time-based lerp instead of fixed factor. Track `lastFrameTime` and scale the interpolation factor by elapsed time.

#### 1b. Track Map Padding + Size Config
**File:** `bridge/overlays/trackmap.html`, `bridge/control-panel.html`

- Add bottom padding (8-10px) to the canvas container so track path doesn't touch the edge
- Add `scale` setting to CUSTOMIZE_FIELDS for trackmap (range 0.5-1.5, default 1.0)
- Apply scale via CSS `transform: scale(X)` on the overlay panel + adjust window size accordingly
- Alternative: just add a `width` and `height` config that resizes the window

#### 1c. Telemetry Data Layer — Tire Compound + Pit Count
**File:** `bridge/telemetry.js`

New telemetry vars to read:
- `CAR_IDX_TIRE_COMPOUND` — per-car tire compound index (number). Map to display names via session YAML `CarSetup` or hardcoded mapping (0=dry, 1=wet typically)
- Pit count: track per-driver using existing `CAR_IDX_ON_PIT_ROAD` transitions. Increment counter when driver enters pit. Store in `pitStopCount` Map.

Add to standings broadcast per driver:
```js
tireCompound: tireCompounds[i] || 0,
pitStops: pitStopCounts.get(i) || 0,
```

Also add gap to leader calculation:
- For each driver, compute time gap to class P1 using `estTime` or `lapDistPct` difference * leader lap time
- Add `gapToLeader` field to standings entries

### Phase 2 — Overlay Improvements

#### 2a. Standings — New Columns
**File:** `bridge/overlays/standings.html`, `bridge/control-panel.html`

Add three new columns to `STANDINGS_COLUMNS`:
- `{ id: 'tire', label: 'Tire', width: 'auto' }` — shows compound icon/letter (D/W/I)
- `{ id: 'pits', label: 'Pits', width: 'auto' }` — shows pit stop count
- `{ id: 'gapLeader', label: 'Gap', width: 'auto' }` — shows +X.Xs gap to class leader

Render:
- Tire: colored dot or letter (green=dry, blue=wet, orange=intermediate)
- Pits: simple number, gray if 0
- Gap: "+1.2s" format, "Leader" for P1

#### 2b. Relative — Tire Compound
**File:** `bridge/overlays/relative.html`, `bridge/control-panel.html`

Add `tire` column to `RELATIVE_COLUMNS`. Same rendering as standings.

#### 2c. Fuel Calculator — Pit Window + Fuel Save
**File:** `bridge/overlays/fuel.html`

Add two new data rows:
- **Pit in X laps**: `lapsOfFuel` rounded down — when fuel runs out, you must pit. Color: green if >5, yellow if 3-5, red if ≤2.
- **Fuel save delta**: Compare avg fuel/lap vs min fuel/lap from history. Show time cost: `(avgLapTime_normal - avgLapTime_saving) ≈ X.Xs/lap`. This is approximate — show as informational.

#### 2d. Session Laps — P/Q/R Tabs
**File:** `bridge/overlays/stintlaps.html`

Add tab bar below header: `ALL | P | Q | R`
- ALL: shows all laps (current behavior)
- P: filter to practice laps only
- Q: filter to qualify laps only  
- R: filter to race laps only

Store all laps across session changes (don't clear). Each lap already has `session` field. Tab filters the display. Summary stats (best/last/avg) recalculate per active filter.

### Phase 3 — New Overlays

#### 3a. Pit Strategy Overlay — "What If I Pit Now?"
**File:** `bridge/overlays/pitstrategy.html` (new)

Shows projected position after a pit stop:
- Current position: P5
- Estimated pit time loss: 28.3s (from pittimes.json)
- Projected position: P8 (calculated by finding which cars would pass during pit)
- Cars that would pass: shows names + gaps

Calculation:
1. Get focus driver's current gap to cars behind (from relative data)
2. Pit time loss = `classPitDeltas[focusClass].avgDelta`
3. Any car behind with gap < pitTimeLoss will pass → count them
4. New position = current position + cars that pass

Subscribe to: `standings`, `session` (for pit deltas)

Size: ~250x200px. Compact card format.

#### 3b. Lap Comparison — Gap Chart
**File:** `bridge/overlays/lapcompare.html` (new)

F1 TV battle mode-style gap chart:
- Line graph: X = lap number, Y = gap in seconds
- Two lines: focus driver vs comparison driver
- Gap grows when comparison driver is faster, shrinks when focus is faster
- Toggle buttons on overlay: "vs Ahead" / "vs Behind" to switch comparison target

Data source: Track `lastLap` per driver per lap (build lap time history per carIdx). Calculate cumulative time difference lap-by-lap.

Subscribe to: `standings` (track lap completions + times)

Size: ~350x200px. Canvas-based chart.

## Data Flow

```
telemetry.js reads:
  CAR_IDX_TIRE_COMPOUND → tireCompounds[] 
  CAR_IDX_ON_PIT_ROAD transitions → pitStopCounts Map
  estTime/lapDistPct → gapToLeader calculation

standings broadcast adds:
  tireCompound, pitStops, gapToLeader per driver

session broadcast adds:
  pitDeltas (already done), fuelPerLap, fuelCapacity (already done)

Overlays consume via WebSocket channels:
  standings → tire, pits, gap columns + pit strategy + lap compare
  session → pit deltas for strategy overlay
```
